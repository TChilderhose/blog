Resume Revamp

2022-12-02   4-minute read   resume • hugo

We’ve currently been hiring at my work and I’ve seen about 200+ resumes in a short amount of time. I made the realization that the visuals of resume effected me more than I thought it should. It made me realize it might be time update and upgrade my resume.

Current Resume Setup

I essentially had 3 resumes that I have been keeping up to date.

  1. The “real” resume that I would submit to places. Google Document with pdf print.
  2. Linkedin experience.
  3. The resume link on this site.

This took a bit of effort to all keep updated and in sync which was partially the reason it’s fallen to the wayside.

Ideally, I would like a single place I can update and use; but I wont be able to automate the LinkedIn resume. But I would like to some how combine the resume on this site and my “real” resume, with the idea of being to just print my resume page to pdf and having it look good.

Hugo Resume Theme Intial Setup

One catch for using a seperate theme for my resume is that Hugo only supports one theme per site.

To get around this I am compiling essentailly 2 sites with their own configs and packaging them together.

./bin/hugo --config=config.yaml --minify
./bin/hugo --config=config_resume.yaml --minify

While ensuring their baseURL are set correctly.

config.yaml

baseURL: '/'

config_resume.yaml

baseURL: "https://tchilderhose.ca/resume/"

This now allows me to have my main blog website theme apply everywhere except for /resume/ which works just how I want.

Hugo Resume Data and Theming

The next thing was to work on organizing some of the data into files before I start trying theming adjustments. I settled on 4 yaml files that I will use to organize my data that will populate my resume.

features.yaml - General (mostly) non-professional information about me.

experience.yaml - All of my work experience in a very easy and flexible way to maintain my resume.

education.yaml - Education information.

projects.yaml - Any non-professional projects that I work on.

Very happy with how the structure turned out.

Hugo Resume Printing

Now to focus on the main reason of this switch, printing.

Luckily there is a css tag for style specific things for print and overall not a lot is needed to get it looking good.

  1. Need to set the page size.
  2. Need to hide the header and footer information from being printed.
  3. Make sure different elements are not being half printed on one page and then the next.

This is the entirety of the css file for print tweaks.

@media print {  
  @page {
    width: 8.5in;
    height: 11in;
    margin: 0.3in 0;
  }

  @page :first {
    margin-top: 0;
  }

  #paper {
    -moz-box-shadow: none;
    -webkit-box-shadow: none;
    box-shadow: none;
    margin: 0;
    padding: 0;
  }

  .item {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  footer {
    display: none;
  }
}

There is still a few tweaks I would like to fix and add when I have more time, but I think it does a really good job.

New Resume Flow

So now my new flow for my resume is to update the yaml files, deploy the site, copy and paste the sections into LinkedIn and when needed, just open the resume page in my browser and save it as a pdf.

Very happy with the new style and streamlined flow of my resume.
